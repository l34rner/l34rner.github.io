postMessage API简记
发表于 2020-01-12 | 更新于 2020-01-12

HTML5通过域的概念对域安全进行了阐明和改进，域是网络上用来建立信任关系的地址的标准。PostMessage API是用来支持跨域通信的一种办法。语法如下：

targetWindow：要接受消息的目标窗口的引用，比如用window.open返回的值，或者是iframe的contentWindow。
message：要发送的数据。
targetOrigin：要发送的目的地址，可以是*号，表示无限制；也可以是一个具体的URI。
接受信息的窗口通过onmessage或者添加对message事件的监听器来处理接受到的消息。

所以，一旦接受消息的窗口不校验信息的来源，同时又用eval处理传递过来的信息，就有可能造成XSS，又或者postMessage的目标地址是个*号，且包含敏感信息的页面中目标窗口的引用可控，就有可能造成信息泄露。
