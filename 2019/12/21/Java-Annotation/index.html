Java Annotation
发表于 2019-12-21 | 更新于 2019-12-21
学习篇·化无炼有

概述

注解不是程序本身，可以对程序作出解释，可以被其他程序读取，比如编译器。注解的格式：@注解名称，还可以添加一些参数值，比如：@SupressWarning(value="unchecked")。

内置注解

@Override：定义在java.lang.Override中，该注解只能用在方法上。如果一个类的某方法上存在@Override，表示该方法是在重写父类中的方法。
@Deprecated：定义在java.lang.Deprecated中，该注解可用于方法、属性、类。表示不推荐程序员使用这样的元素，通常是因为它很危险或者存在更好的选择。
@SuppressWarnings：定义在java.lang.SuppresWarnings中，用来抑制编译时的警告信息，该注解需要配合参数使用，这些参数都是预先定义好的，选择性使用即可。

  • @SuppressWarnings(“all”)
  • @SuppressWarnings(“unchecked”)
  • @SuppressWarnings(value={“unchecked”,”deprecation”})

元注解

元注解就是注解其他注解的注解。即对其他的注解做出解释说明。java定义了4个元注解，在java.lang.annotation中。
@Target：表示一个注解可以被用在哪些地方（即：是否可以在类、方法、属性等处使用）
@Retention：表示一个注解的生命周期（source<class<runtime）。source表示注解只在源代码阶段有效；class表示注解只被保留到编译阶段，不会被加载到JVM中；runtime表示注解一直到运行期都有效，会被加载到JVM中。
@Document：将注解的元素包含到javadoc中
@Inherited：表示子类可以继承父类的注解
